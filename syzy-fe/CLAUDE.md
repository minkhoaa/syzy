## Commands

```bash
pnpm dev                  # Start Next.js dev server (port 3000, uses --webpack flag)
pnpm build                # Production build
pnpm start                # Start production server (port 3000)
pnpm lint                 # ESLint (flat config: eslint.config.mjs, custom globalIgnores override)
pnpm test                 # Run tests (vitest run)
pnpm test:watch           # Run tests in watch mode (vitest)
npx vitest run __tests__/path/to/file.test.ts  # Run a single test file
pnpm generate:api         # Regenerate API client from backend OpenAPI spec (backend must be running)
pnpm generate:api:dev     # Same but keeps temp files for debugging Kubb output
```

The `--webpack` flag on `dev` and `build` is required because Turbopack cannot handle snarkjs WASM loading.

The `generate:api` pipeline: `fix-openapi.js` → `kubb` → `fix-imports.js` → `clean-temp.js`. Fetches the spec from `NEXT_PUBLIC_API_URL/api-json` (default `http://localhost:7777/api-json`). Output goes to `lib/api-client/` with subdirectories `client/`, `hooks/`, `types/`, `schemas/`. Generated code imports the custom Axios client from `@/lib/kubb`.

### Testing

Vitest v4, node environment, globals enabled (`describe`/`it`/`expect` available without imports). Tests live in `__tests__/**/*.test.ts`. The `@/` path alias works in tests. Both `contracts/` and `node_modules/` are excluded.

## Architecture

**Next.js 16 App Router** (React 19, Tailwind v4 CSS-first config via `@tailwindcss/postcss`) for a Solana prediction market platform.

### Feature-Based Organization

Business logic is organized by domain in `features/`, **not** in a flat `hooks/` or `store/` directory:

```
features/
  auth/         → hooks/use-wallet-auth, use-reown-wallet; store/use-auth-store
  trading/      → hooks/use-prediction-market (~2500 lines), use-negrisk, use-switchboard-feed, use-wallet-transaction
  privacy/      → hooks/use-zk (~39KB), use-tee, use-zk-notes, use-shielded-positions
  markets/      → hooks/use-market-list, use-market-detail, use-db-market, use-backend-markets, use-market-news, use-market-snapshots,
                  use-multi-market-detail, use-multi-market-snapshots, use-recent-trades, use-kalshi-events, use-polymarket-events
                  store/use-market-list-store; utils/
  portfolio/    → hooks/use-portfolio, use-watchlist
  staking/      → hooks/use-staking
  admin/        → hooks/use-admin-metrics, use-admin-operations, use-is-admin, use-oracle-feeds
                  store/use-oracle-feed-store
  analytics/    → hooks/use-metrics, use-coingecko, use-pump-token-ohlcv
  content/      → hooks/use-articles, use-blogs, use-comments
  onboarding/   → hooks/use-tour; store/use-tour-store
```

The top-level `hooks/` directory has utility hooks (`use-agent-session`, `use-chat-history`, `use-mobile`) and a `common/` subdirectory.

### Route Groups

- `(landing)/` — Public marketing pages + blog
- `(dashboard)/` — Authenticated app (dashboard, markets, portfolio, history, settings, shield)
- `(dashboard)/markets/[slug]/` — Market/event group detail pages (NegRisk multi-outcome). Note: `/events/:slug` permanently redirects to `/markets/:slug` via `next.config.ts`
- `(admin)/` — Admin panel (protected by `RequireAdmin` component wrapper)
- `(agent)/` — AI agent interface
- `(x402)/` — x402 paid API docs, SDK, and examples
- `staking/` — Staking page (standalone)
- `faucet/` — Token faucet

Route protection is at the component level (`components/auth/require-auth.tsx`, `require-admin.tsx`), not via middleware. No `middleware.ts` exists.

### Provider Chain (root layout)

`ThemeProvider` → `QueryProvider` → `ReownProvider` → `[AutoLogin, TooltipProvider(children), Toaster]`

`GoogleAnalyticsProvider` and `VercelAnalyticsProvider` are outside the chain (directly in body).

`snarkjs` v0.7.5 loaded via CDN `<Script strategy="beforeInteractive">` (not bundled). Accessed as `window.snarkjs` in `features/privacy/lib/`.

Fonts: `Inter` (`--font-sans` on `<html>`), `Geist` + `Geist_Mono` (on `<body>`) via `next/font/google`.

### Zustand Stores

- `features/auth/store/use-auth-store.ts` — Persisted to localStorage `'oyrade-auth-storage'`
- `features/markets/store/use-market-list-store.ts` — In-memory
- `features/admin/store/use-oracle-feed-store.ts` — Persisted to localStorage `'oyrade-oracle-feeds'`
- `features/onboarding/store/use-tour-store.ts` — Tour state

### Wallet & Auth Flow

1. Connect wallet via Reown AppKit (Phantom + Solflare)
2. Request nonce from backend (`POST /auth/nonce`)
3. Sign nonce message with wallet
4. Backend verifies ED25519 signature, returns JWT pair
5. Tokens stored in Zustand auth store (persisted to localStorage)
6. `lib/kubb.ts` Axios interceptor auto-injects auth headers, handles 401 refresh via `POST /api/auth/refresh`, and fires `CustomEvent('auth:token-expired')` on `window` when auth fails

### Solana Program Integration

- **IDLs**: `lib/constants/IDL.json` (prediction market), `lib/constants/REWARD_POOL_IDL.json` (reward pool) — copied from the Anchor contracts repo's `target/idl/` after `anchor build` (contracts live in a separate sibling repo, not in this frontend repo)
- **Program ID**: Configurable via `NEXT_PUBLIC_PROGRAM_ID`, defaults to devnet `HEmsoVhRJT4DRmGqZPk136eSLFPNRi6RmMqFNk4eJsVN`
- **PDA seeds** in `lib/constants/programs.ts` must match the Anchor program's `constants.rs`
- **Key seeds**: `config`, `market`, `global`, `userinfo`, `market_v1_config`, `market_group`, `market_group_config`, `shielded_pool`, `nullifier_shard`, `tee_registry`, `leaves_indexer`, `subtree_indexer`
- **Staking seeds** (in `lib/constants/staking.ts`): `reward_pool`, `reward_vault`, `stake_user`
- **Token constants**: `TOKEN_DECIMALS = 6`, `TOKEN_MULTIPLIER = 10^6`, `MAX_NEGRISK_MARKETS = 10`
- **Enums**: `Outcome { Yes=0, No=1 }`, `Side { Buy=0, Sell=1 }`
- **Transaction pattern**: build instruction → `addComputeBudget(tx, units, fee)` → sign → confirm
- **Two signing paths**: `signAndSendTransaction` vs `signTransaction` + `sendRawTransaction` (depends on wallet capability)
- Type casts for `program.methods` require `as unknown as { ... }` pattern (strict TypeScript)

### NegRisk / Event Groups

Multi-outcome events (e.g., "Who wins the election?") where multiple binary markets are grouped. The `useNegRisk()` hook (`features/trading/hooks/use-negrisk.ts`) provides:
- `groupSplitCollateral` — Deposit SOL to get YES tokens on all N markets in a group
- `groupMergeCollateral` — Burn YES from all markets to get SOL back
- `convertPositions` — NegRisk conversion: burn NO on selected markets → SOL + YES on remaining
- `syncUserInfo` — Sync UserInfo PDA from ATA balances (fixes v1 claim bug)

Uses `remainingAccounts` pattern: 6 accounts per market for convert, 4 per market for split/merge.

### Query Config Presets (`lib/kubb.ts`)

- `$` — Standard (5min stale, 10min gcTime, refetch on focus, retry: 1)
- `$live` — Live data (30s stale, 1min gcTime, 30s refetch interval, retry: 2)
- `$fresh` — No cache (0 stale, 0 gcTime, always refetch on mount)
- `$background` — Background (15min stale, 30min gcTime, no auto-refetch, retry: 0)

### Market Versions

- **v0 (Dynamic Parimutuel):** Constant product AMM (`k = x * y`). Legacy. Instructions: `createMarket`, `swap`, `resolution`, `claimWinnings`
- **v1 (Conditional Token Framework):** `1 YES + 1 NO = 1 SOL collateral`. Instructions: `depositCollateral`, `withdrawCollateral`, `swapV2`, `claimWinningsV2`

### Tailwind v4 CSS-First Config

No `tailwind.config.*` file. Configuration is in `app/globals.css`:
- `@import "tailwindcss"` + `@import "tw-animate-css"` + `@import "shadcn/tailwind.css"`
- `@plugin "@tailwindcss/typography"` for prose styling
- `@custom-variant dark (&:is(.dark *))` — dark mode via class strategy
- `@source not ".claude"` — excludes `.claude/` from Tailwind scanning

## Code Patterns

- **Path alias**: `@/` maps to project root
- **Styling**: Tailwind CSS v4 with `cn()` utility (`clsx` + `tailwind-merge`) from `lib/utils.ts`
- **Icons**: `lucide-react` + `@web3icons/react`
- **Toasts**: `sonner`
- **Forms**: `react-hook-form` + `zod` (v4 — breaking changes from v3)
- **Charts**: `lightweight-charts` (trading), `recharts` (analytics)
- **Rich text**: Tiptap editor (comments, blog posts)
- **Animations**: `framer-motion` / `motion`
- **3D**: `@react-three/fiber` + `@react-three/drei` (Three.js)
- **Smooth scroll**: `lenis`
- **Markdown**: `react-markdown` + `remark-gfm`
- **x402**: `@x402/fetch` + `@x402/svm` for paid API endpoints
- **Onboarding**: `driver.js` for guided tours (CSS imported in `globals.css`, overrides in `features/onboarding/styles/`)
- **Package manager**: pnpm
- **Strict TypeScript** enabled (unlike backend which has `strictNullChecks: false`)
- **Solana SDKs**: Both `@solana/web3.js` v1 and `@solana/kit` v6 are present
- **tsconfig**: `target: ES2017`, `moduleResolution: bundler`, `isolatedModules: true`, `incremental: true`, excludes `ref/` and `docs/`
